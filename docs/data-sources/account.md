# slurm_account (Data Source)

Reads an existing Slurm account by name. Returns the account metadata
(description, organization, parent account) and the limits attached to
its cluster-level association — fairshare, default QOS, allowed QOS,
max running jobs, and TRES limits.

Every attribute is read-only.

## Example Usage

```terraform
# Look up an existing Slurm account by name. Returns its metadata and
# the limits attached to the account-level association — fairshare,
# default QOS, allowed QOS, max running jobs, and TRES limits.

data "slurm_account" "physics" {
  name = "physics"
}

# Sub-account that inherits the parent's allowed QOS list.
resource "slurm_account" "physics_hep" {
  name           = "physics_hep"
  parent_account = data.slurm_account.physics.name
  allowed_qos    = data.slurm_account.physics.allowed_qos
}
```

### Required

- `name` (String) The name of the account to look up.

### Read-Only

- `allowed_qos` (List of String) Allowed QOS list on the account-level association.

- `description` (String) Account description.

- `id` (String) The account name (same as name).
- `max_jobs` (Number) Maximum running jobs on the account-level association (MaxJobs).
- `max_tres_mins_per_job` (Attributes Set) Max TRES-minutes per job (MaxTRESMins). (see [below for nested schema](#nestedatt--max_tres_mins_per_job))
- `max_tres_per_job` (Attributes Set) Max TRES per job (MaxTRES) on the account-level association. (see [below for nested schema](#nestedatt--max_tres_per_job))
