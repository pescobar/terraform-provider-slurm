# slurm_conf (Data Source)

Reads the active slurmctld configuration — the live equivalent of
`scontrol show config`. Useful for preflight assertions (for example checking
that `AccountingStorageEnforce` includes `associations`, which the accounting
model managed by this provider assumes) or for referencing cluster facts such
as the running Slurm version.

~> Requires Slurm **26.05 or later** with `api_version = "v0.0.45"` (or
later). With an older `api_version` the plan fails with an error naming the
required versions; with a new enough `api_version` but an older cluster, the
error points at the server version instead.

## Example Usage

```terraform
# Read the active slurmctld configuration — the live equivalent of
# `scontrol show config`. Requires Slurm 26.05+ (API version v0.0.45+).
data "slurm_conf" "cluster" {}

output "running_slurm_version" {
  value = data.slurm_conf.cluster.slurm_version
}

# Preflight assertion: warn when the cluster is not enforcing associations —
# the accounting model this provider manages assumes it.
check "accounting_enforce_includes_associations" {
  assert {
    condition     = strcontains(data.slurm_conf.cluster.conf["AccountingStorageEnforce"], "associations")
    error_message = "AccountingStorageEnforce must include 'associations'."
  }
}
```

## Schema

### Read-Only

- `cluster_name` (String) ClusterName from the active configuration.
- `conf` (Map of String) All configuration keys, using slurm.conf capitalisation (e.g. `SchedulerType`). Values are flattened to strings: lists are comma-joined, unset tri-state numbers are empty, unlimited values are `infinite`.
- `conf_path` (String) Path of the slurm.conf the controller loaded.
- `id` (String) Always `slurm_conf`.
- `slurm_version` (String) The running Slurm version (e.g. `26.05.1`).
