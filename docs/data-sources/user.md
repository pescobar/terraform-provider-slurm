# slurm_user (Data Source)

Reads an existing Slurm user by name. Returns the user's admin level,
default account (derived from the association marked `is_default`),
and every association as a nested `association` block — same shape as
the `slurm_user` resource accepts on input.

Every attribute is read-only.

## Example Usage

```terraform
# Look up an existing Slurm user by name. Returns the user's admin level,
# default account, and every association with its full limit set as a
# nested block — exactly the shape slurm_user accepts on input.

data "slurm_user" "alice" {
  name = "alice"
}

# Use the data source's default_account in another resource. Useful in
# multi-team setups where the user is managed elsewhere but the account
# binding needs to be referenced.
output "alice_default_account" {
  value = data.slurm_user.alice.default_account
}

# All accounts the user has an association with.
output "alice_accounts" {
  value = [for a in data.slurm_user.alice.association : a.account]
}
```

## Schema

### Required

- `name` (String) The Slurm user name to look up.

### Read-Only

- `admin_level` (String) Admin level: None, Operator, or Administrator.
- `association` (Attributes Set) All associations for this user, one entry per (account, partition) pair. Every field is read-only. (see [below for nested schema](#nestedatt--association))
- `default_account` (String) The user's default Slurm account, derived from the association marked is_default.
- `default_wc_key` (String) Default workload characterization key.
- `id` (String) The user name (same as name).
