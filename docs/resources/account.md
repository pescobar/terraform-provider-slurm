# slurm_account (Resource)

Manages a Slurm account and its account-level association (limits, QOS, fairshare).

## Example Usage

```terraform
resource "slurm_account" "physics" {
  name         = "physics"
  description  = "Physics department"
  organization = "university"
  fairshare    = 100
  default_qos  = slurm_qos.standard.name
  allowed_qos  = [slurm_qos.standard.name, slurm_qos.priority.name]
}

# Child account inheriting from physics
resource "slurm_account" "hep" {
  name           = "hep"
  description    = "High Energy Physics group"
  organization   = "university"
  parent_account = slurm_account.physics.name
  fairshare      = 50
}

# Account with per-job and group TRES limits
resource "slurm_account" "gpu_users" {
  name        = "gpu_users"
  description = "GPU users group"

  # Per-job limits
  max_tres_per_job = [
    { type = "cpu", count = 64 },
    { type = "mem", count = 131072 }, # 128 GB in MB
    { type = "gres", name = "gpu", count = 8 },
  ]

  max_tres_per_node = [
    { type = "gres", name = "gpu", count = 4 },
  ]

  # Group-aggregate limits shared across all running jobs in this account
  grp_tres = [
    { type = "gres", name = "gpu", count = 32 },
  ]

  grp_tres_mins = [
    { type = "gres", name = "gpu", count = 460800 }, # 32 GPUs × 14400 min (10 d)
  ]

  grp_tres_run_mins = [
    { type = "gres", name = "gpu", count = 230400 }, # cap on currently-running GPU·min
  ]
}
```

## Import

> **Bulk import tip:** [`tools/generate_import/generate_import.py`](https://github.com/pescobar/terraform-provider-slurm/blob/master/tools/generate_import/generate_import.py)
> queries a running `slurmrestd` instance and generates ready-to-use HCL files plus
> `import {}` blocks for all QOS, accounts, and users in one shot. See its
> [README](https://github.com/pescobar/terraform-provider-slurm/blob/master/tools/generate_import/README.md)
> for usage.

Import an account by its Slurm name:

```bash
tofu import slurm_account.physics physics
```

### Null-preservation after import

`slurm_account` uses **Optional-only** schema fields (no `Computed`). The provider
only writes a field into state when it is already non-null in prior state. This
prevents Slurm's inherited defaults — such as `fairshare = 1` or a QOS list
inherited from the parent account — from appearing as drift after a fresh import.

After `tofu import`, Optional fields (`fairshare`, `default_qos`, `allowed_qos`,
etc.) will be null in state. A **reconcile apply** is required to write
config-declared values to Slurm and populate state. After that, `tofu plan` must
be clean.

### Standard import workflow

```bash
# Step 1 — import (Optional fields are null in state)
tofu import slurm_account.physics physics

# Step 2 — reconcile apply (writes config-declared values to Slurm)
tofu apply

# Step 3 — verify clean plan (must report no changes)
tofu plan -detailed-exitcode
```
